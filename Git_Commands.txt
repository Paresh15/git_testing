VERSION CONTROL SYSTEM:
GIT PHASES::
CLONE : Download the code
ADD   : From working to staging
COMMIT: Staging to loacal repo
PUSH  : Local repo to remote repo
PULL  : From remote repo to local repo (fetch(down)+merge(save))
MERGE : Saving the code
FETCH : Download the code

GIT COMMANDS::
git init         --> initilize the empty repo(to store snap)
git status       --> to see the items (Working dir)
git add test.txt (Staging area)
git commit -m "this is my  test file" (local repo)
git add -A       (All changes are upload to staging)
git log          (Show all changes)
git reflog       (Changes in short)

copy the git remote add origin (For linking)
git remote -v (Show URL)
git push (will fail)
git push --set-upstream origin master
git clone URL
.  (. for clonning)
